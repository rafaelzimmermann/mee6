# React + REST API Refactoring Plan

## Executive Summary

Refactor mee6 from a server-side rendered FastAPI application with Jinja2 templates to a modern React Single Page Application (SPA) with a pure JSON REST API backend. This transformation will maintain all existing functionality while providing complete separation between frontend and backend.

## Current Architecture

### Backend
- FastAPI returning HTML templates (Jinja2)
- 4 route modules: pipelines, triggers, integrations, history
- Pydantic models for basic validation
- Repository pattern for database access
- Only 2 JSON endpoints exist

### Frontend
- Server-side rendered HTML (8 main templates + 4 partials)
- Vanilla JavaScript with manual state management
- HTMX for real-time updates (polling)
- Mixed form submission patterns
- HTML5 validation only

## Proposed Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    React SPA (Frontend)                      │
│  - React Components + React Router + React Query           │
│  - Client-side validation (Zod)                           │
│  - State management (Zustand or React Context)            │
└──────────────────────┬──────────────────────────────────────────┘
                       │ REST API (JSON only)
                       │
┌──────────────────────▼──────────────────────────────────────────┐
│                   FastAPI Backend                            │
│  - Pure JSON endpoints                                     │
│  - Enhanced Pydantic validation                            │
│  - Repository layer unchanged                                │
│  - WebSocket/SSE for real-time updates                     │
└───────────────────────────────────────────────────────────────────┘
```

### 1.2 Convert All Routes to JSON API

**File: `mee6/web/api/pipelines.py`**
```python
from fastapi import APIRouter, HTTPException, status
from mee6.web.api.models import *
from mee6.db.repository import PipelineRepository, TriggerRepository
from mee6.pipelines.store import pipeline_store
from mee6.pipelines.plugin_registry import AGENT_PLUGINS

router = APIRouter()

@router.get("/api/v1/pipelines", response_model=List[PipelineResponse])
async def list_pipelines():
    pipelines = await pipeline_store.list()
    return [p.model_dump() for p in pipelines]

@router.get("/api/v1/pipelines/{pipeline_id}", response_model=PipelineResponse)
async def get_pipeline(pipeline_id: str):
    pipeline = await pipeline_store.get(pipeline_id)
    if not pipeline:
        raise HTTPException(status_code=404, detail="Pipeline not found")
    return pipeline.model_dump()

@router.post("/api/v1/pipelines", response_model=PipelineCreateResponse, status_code=201)
async def create_pipeline(data: PipelineCreateRequest):
    pipeline_id = str(uuid.uuid4())
    pipeline = Pipeline(
        id=pipeline_id,
        name=data.name,
        steps=[PipelineStep(**step) for step in data.steps],
    )
    await pipeline_store.upsert(pipeline)
    return PipelineCreateResponse(id=pipeline_id, message="Pipeline created")

@router.put("/api/v1/pipelines/{pipeline_id}", response_model=PipelineResponse)
async def update_pipeline(pipeline_id: str, data: PipelineCreateRequest):
    existing = await pipeline_store.get(pipeline_id)
    if not existing:
        raise HTTPException(status_code=404, detail="Pipeline not found")

    pipeline = Pipeline(
        id=pipeline_id,
        name=data.name,
        steps=[PipelineStep(**step) for step in data.steps],
    )
    await pipeline_store.upsert(pipeline)
    scheduler.update_pipeline_name(pipeline_id, pipeline.name)
    return pipeline.model_dump()

@router.delete("/api/v1/pipelines/{pipeline_id}", status_code=204)
async def delete_pipeline(pipeline_id: str):
    async with AsyncSessionLocal() as session:
        if await TriggerRepository(session).exists_for_pipeline(pipeline_id):
            raise HTTPException(
                status_code=400,
                detail="Cannot delete: pipeline has associated triggers"
            )
    await pipeline_store.delete(pipeline_id)
```

```typescript
import { z } from 'zod';

export const pipelineSchema = z.object({
  name: z.string()
    .min(1, 'Pipeline name is required')
    .max(100, 'Pipeline name must be less than 100 characters'),
  steps: z.array(z.object({
    agent_type: z.string().min(1, 'Agent type is required'),
    config: z.record(z.string()),
  })).min(1, 'Pipeline must have at least one step'),
});

export const memoryConfigSchema = z.object({
  label: z.string()
    .min(1, 'Label is required')
    .max(50, 'Label must be less than 50 characters'),
  max_memories: z.number().int().positive().default(20),
  ttl_hours: z.number().int().positive().default(720),
  max_value_size: z.number().int().positive().default(2000),
});

// Shared schemas for server/client
export type PipelineInput = z.infer<typeof pipelineSchema>;
```

```typescript
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '@/lib/api';
import { pipelineSchema } from '@/lib/validation';

export function usePipelines() {
  return useQuery({
    queryKey: ['pipelines'],
    queryFn: api.pipelines.list,
  });
}

export function usePipeline(id: string) {
  return useQuery({
    queryKey: ['pipelines', id],
    queryFn: () => api.pipelines.get(id),
    enabled: !!id,
  });
}

export function useCreatePipeline() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (data: PipelineInput) => api.pipelines.create(data),
    onSuccess: () => {
      queryClient.invalidateQueries(['pipelines']);
    },
  });
}

export function useUpdatePipeline() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: ({ id, data }: { id: string; data: PipelineInput }) =>
      api.pipelines.update(id, data),
    onSuccess: () => {
      queryClient.invalidateQueries(['pipelines']);
    },
  });
}

export function useDeletePipeline() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (id: string) => api.pipelines.delete(id),
    onSuccess: () => {
      queryClient.invalidateQueries(['pipelines']);
    },
  });
}
```

### 2.6 Pipeline Form Component

**File: `mee6/frontend/src/components/pipelines/PipelineForm.tsx`**
```typescript
import { useState } from 'react';
import { useNavigate, useParams } from 'react-router-dom';
import { usePipeline, useUpdatePipeline, useCreatePipeline, useAgents } from '@/hooks';
import { pipelineSchema } from '@/lib/validation';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { StepCard } from './StepCard';

export function PipelineForm() {
  const { id } = useParams();
  const navigate = useNavigate();
  const { data: pipeline, isLoading } = usePipeline(id);
  const { data: agents } = useAgents();
  const updatePipeline = useUpdatePipeline();
  const createPipeline = useCreatePipeline();

  const {
    register,
    handleSubmit,
    formState: { errors },
  } = useForm({
    resolver: zodResolver(pipelineSchema),
    defaultValues: pipeline || { name: '', steps: [{ agent_type: '', config: {} }] },
  });

  const onSubmit = (data: PipelineInput) => {
    if (id) {
      updatePipeline.mutate({ id, data });
    } else {
      createPipeline.mutate(data);
    }
    navigate('/pipelines');
  };

  if (isLoading) return <div>Loading...</div>;

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <Layout title={id ? 'Edit Pipeline' : 'New Pipeline'}>
        <Input label="Name" {...register('name')} error={errors.name?.message} />

        <div className="space-y-4">
          {pipeline?.steps?.map((step, idx) => (
            <StepCard
              key={idx}
              index={idx}
              agents={agents}
              defaultAgent={step.agent_type}
              defaultConfig={step.config}
            />
          ))}
        </div>

        <div className="flex gap-2">
          <Button type="submit">Save Pipeline</Button>
          <Button type="button" variant="secondary" onClick={() => navigate('/pipelines')}>
            Cancel
          </Button>
        </div>
      </Layout>
    </form>
  );
}
```

### 2.7 Real-Time Updates Hook

**File: `mee6/frontend/src/hooks/useRealtime.ts`**
```typescript
import { useEffect, useState } from 'react';
import { useQueryClient } from '@tanstack/react-query';

export function useRealtime() {
  const queryClient = useQueryClient();
  const [eventSource, setEventSource] = useState<EventSource | null>(null);

  useEffect(() => {
    const es = new EventSource('/api/v1/events/history');
    setEventSource(es);

    es.onmessage = (event) => {
      const data = JSON.parse(event.data);
      if (data.type === 'history_update') {
        queryClient.invalidateQueries(['history']);
      }
      if (data.type === 'running_update') {
        queryClient.invalidateQueries(['running']);
      }
    };

    return () => es.close();
  }, [queryClient]);

  return eventSource;
}
```

## Phase 4: Migration Strategy

### 4.1 Incremental Migration (Recommended)

**Week 1-2: Backend API Foundation**
1. Create API response models in `mee6/web/api/models.py`
2. Implement `/api/v1/pipelines` endpoints
3. Add CORS middleware
4. Test with curl/Postman

**Week 3-4: Frontend Pipeline Management**
1. Set up React + Vite project
2. Implement PipelineList component
3. Implement PipelineForm component
4. Add routing
5. Test pipeline CRUD

**Week 5-6: Remaining API + Frontend Features**
1. Implement triggers API and components
2. Implement history API and components
3. Implement integrations API and components
4. Add real-time updates via SSE

**Week 7-8: Polish & Deploy**
1. Add comprehensive validation (Zod + Pydantic)
2. Implement error handling
3. Add loading states
4. Deploy with nginx reverse proxy
5. Remove old template routes

### 4.2 Big Bang Migration (Alternative)

1. Complete all API endpoints first
2. Build full React frontend
3. Single deployment cutover
4. Keep backend as fallback during transition
