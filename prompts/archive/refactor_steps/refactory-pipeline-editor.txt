# Refactoring Plan: Pipeline Editor (mee6/web/templates/pipeline_editor.html)

## Current Issues

### 1. If-else Mass in renderFields() Function (Lines 51-116)
- 7 different conditional branches for field types
- Inline HTML generation with string concatenation
- Complex nested conditions for select-based fields
- No code reuse between similar field types

### 2. Mixed Concerns
- Rendering logic mixed with DOM manipulation
- State management scattered across global variables
- Event handling interwoven with business logic
- Validation tightly coupled to DOM

### 3. No Separation of Concerns
- All JavaScript is inline in HTML template
- No modularization
- Difficult to test individual components
- Hard to maintain and extend

### 4. Lack of Reusability
- Field rendering logic is not reusable
- No component-based architecture
- Duplication in similar field types (select, group_select, calendar_select)

### 5. Insufficient Field Validation
- Only validates memory agent checkbox state (validateMemorySteps function)
- No validation for required fields with missing values
- No validation for empty text inputs
- No validation for select fields with no option selected
- Allows saving pipelines with completely empty steps
- No client-side feedback for validation errors
- No validation on field change, only on save

### 6. No Testing Infrastructure
- JavaScript code is not testable
- No test coverage for client-side logic
- Difficult to verify correctness of dynamic field rendering
- No tests for validation logic

## Proposed Architecture

#### Module Responsibilities

**state-manager.js**
- Manage pipeline state (name, steps)
- Handle schema cache
- Provide reactive state updates
- Centralized state mutations

**field-renderer.js**
- Coordinate field component rendering
- Handle placeholder hints injection
- Manage field container updates

**field-components/** (Strategy Pattern)
- Each file exports a render function
- Consistent interface: `render(field, value, index, options)`
- Component registry maps field_type to renderer

**step-manager.js**
- Add, remove, move steps
- Update step data
- Reorder steps
- Step index management

**validator.js**
- Validate individual fields
- Validate memory agent steps
- Return error messages

**api-client.js**
- Fetch schemas
- Save/update pipelines
- Handle errors

export function render(field, value, index, placeholderHints) {
  const id = `field-${index}-${field.name}`;
  const placeholderEsc = esc(field.placeholder);
  const valueEsc = esc(value || '');
  const required = field.required ? 'required' : '';

  let html = `<label class="field-label">${esc(field.label)}`;
  html += `<textarea name="${esc(field.name)}" id="${id}" placeholder="${placeholderEsc}" ${required}>${valueEsc}</textarea>`;

  if (placeholderHints.length) {
    const hints = placeholderHints.map(p => `<code>${esc(p)}</code>`).join(' ');
    html += `<span class="field-hint">Placeholders: ${hints}</span>`;
  }

  html += `</label>`;
  return html;
}

import { renderSelectOptionsWithHint } from '../utils/select-utils.js';

export function render(field, value, index) {
  const id = `field-${index}-${field.name}`;
  const hintId = `ghint-${index}-${esc(field.name)}`;
  const required = field.required ? 'required' : '';

  const handleChange = `(function(s){
    var o=s.options[s.selectedIndex];
    var h=document.getElementById('${hintId}');
    if(h)h.textContent=o?o.dataset.hint:'';
  })(this)`;

  let html = `<label class="field-label">${esc(field.label)}`;
  html += `<select name="${esc(field.name)}" id="${id}" ${required} onchange="${handleChange}">`;

  html += `<option value="">— select group —</option>`;
  let hintVal = '';

  for (const opt of field.options) {
    const parts = opt.split('||', 2);
    const jid = parts.length > 1 ? parts[1] : parts[0];
    const selected = value === jid ? ' selected' : '';
    if (value === jid) hintVal = jid;
    html += `<option value="${esc(jid)}" data-hint="${esc(jid)}"${selected}>${esc(parts[0])}</option>`;
  }

  html += `</select>`;
  html += `<span class="field-hint" id="${hintId}">${esc(hintVal)}</span>`;
  html += `</label>`;

  return html;
}

### Phase 4: State Management Refactoring

#### Centralized State Store
```javascript
// modules/state-manager.js
class PipelineEditorState {
  constructor() {
    this.pipeline = {
      id: null,
      name: '',
      steps: []
    };
    this.schemas = {};
    this.listeners = [];
  }

  setPipeline(pipeline) {
    this.pipeline = { ...this.pipeline, ...pipeline };
    this.notify('pipeline-updated');
  }

  setSchemas(schemas) {
    this.schemas = schemas;
    this.notify('schemas-loaded');
  }

  addStep(stepData) {
    this.pipeline.steps.push({
      agent_type: '',
      config: {},
      ...stepData
    });
    this.notify('steps-updated');
  }

  removeStep(index) {
    this.pipeline.steps.splice(index, 1);
    this.notify('steps-updated');
  }

  moveStepUp(index) {
    if (index > 0) {
      const temp = this.pipeline.steps[index];
      this.pipeline.steps[index] = this.pipeline.steps[index - 1];
      this.pipeline.steps[index - 1] = temp;
      this.notify('steps-updated');
    }
  }

  subscribe(event, callback) {
    this.listeners.push({ event, callback });
  }

  notify(event, data) {
    this.listeners
      .filter(l => l.event === event)
      .forEach(l => l.callback(data));
  }
}

export const state = new PipelineEditorState();
```

import { state } from './state-manager.js';

const handlers = {
  agentChange: async (card, stepIndex) => {
    const select = card.querySelector('.agent-select');
    state.updateStep(stepIndex, {
      agent_type: select.value,
      config: {}
    });
  },

  removeStep: async (card, stepIndex) => {
    state.removeStep(stepIndex);
  },

  moveUp: async (stepIndex) => {
    state.moveStepUp(stepIndex);
  },

  moveDown: async (stepIndex) => {
    state.moveStepDown(stepIndex);
  }
};

export function setupEventDelegation(container) {
  container.addEventListener('change', (e) => {
    if (e.target.classList.contains('agent-select')) {
      const card = e.target.closest('.step-card');
      const stepIndex = parseInt(card.dataset.idx);
      handlers.agentChange(card, stepIndex);
    }
  });

  container.addEventListener('click', (e) => {
    const card = e.target.closest('.step-card');
    if (!card) return;
    const stepIndex = parseInt(card.dataset.idx);

    if (e.target.classList.contains('remove-step')) {
      handlers.removeStep(card, stepIndex);
    } else if (e.target.classList.contains('move-up')) {
      handlers.moveUp(stepIndex);
    } else if (e.target.classList.contains('move-down')) {
      handlers.moveDown(stepIndex);
    }
  });
}

### Phase 6: Validation Refactoring

#### Current Validation Gaps

**Issues Identified:**
1. `validateMemorySteps()` only checks memory_agent checkboxes
2. No validation for required text/textarea fields being empty
3. No validation for required select/combobox fields having no selection
4. No validation for empty pipeline (no steps)
5. No validation for steps without agent type selected
6. No real-time validation feedback (only on save)
7. Validation errors shown via alert() instead of inline UI
8. No visual indication of which fields have errors

**Critical Bug Example:**
- User can create a pipeline with steps that have no values
- User can save pipeline with empty required fields
- User can submit pipeline without any agent type selected
- Backend accepts invalid pipelines

// modules/validator.js
export class ValidationError {
  constructor(fieldName, stepIndex, fieldType, message) {
    this.fieldName = fieldName;
    this.stepIndex = stepIndex;
    this.fieldType = fieldType;
    this.message = message;
  }
}

// Field-level validators for different types
const fieldValidators = {
  text: (field, value) => {
    if (field.required && (!value || value.trim() === '')) {
      return `${field.label} is required`;
    }
    return null;
  },

  textarea: (field, value) => {
    if (field.required && (!value || value.trim() === '')) {
      return `${field.label} is required`;
    }
    return null;
  },

  select: (field, value) => {
    if (field.required && (!value || value === '')) {
      return `Please select a ${field.label.toLowerCase()}`;
    }
    return null;
  },

  combobox: (field, value) => {
    if (field.required && (!value || value.trim() === '')) {
      return `${field.label} is required`;
    }
    return null;
  },

  group_select: (field, value) => {
    if (field.required && (!value || value === '')) {
      return `Please select a ${field.label.toLowerCase()}`;
    }
    return null;
  },

  calendar_select: (field, value) => {
    if (field.required && (!value || value === '')) {
      return `Please select a ${field.label.toLowerCase()}`;
    }
    return null;
  },

  checkbox: (field, value) => {
    // Checkboxes are optional by default unless required is true
    // For checkboxes, 'required' typically means it must be checked
    if (field.required && value !== 'on') {
      return `${field.label} must be checked`;
    }
    return null;
  }
};

export function validateField(field, value) {
  const validator = fieldValidators[field.field_type] || fieldValidators.text;
  return validator(field, value);
}

export function validateStepConfig(step, stepIndex, schema) {
  const errors = [];

  if (!step.agent_type) {
    errors.push(new ValidationError('agent_type', stepIndex, 'select', 'Agent type is required'));
    return errors;
  }

  const fields = schema[step.agent_type] || [];
  for (const field of fields) {
    const value = step.config[field.name];
    const errorMessage = validateField(field, value);
    if (errorMessage) {
      errors.push(new ValidationError(
        field.name,
        stepIndex,
        field.field_type,
        errorMessage
      ));
    }
  }

  // Special validation for memory agent
  if (step.agent_type === 'memory_agent') {
    const hasRead = step.config.read_memory === 'on';
    const hasWrite = step.config.write_memory === 'on';
    if (!hasRead && !hasWrite) {
      errors.push(new ValidationError(
        'memory_options',
        stepIndex,
        'checkbox',
        'Please select at least one of Read Memory or Write Memory'
      ));
    }
  }

  return errors;
}

export function validateStep(step, stepIndex, schema) {
  const errors = validateStepConfig(step, stepIndex, schema);
  return errors;
}

export function validatePipeline(pipeline, schema) {
  const errors = [];

  // Validate pipeline has at least one step
  if (!pipeline.steps || pipeline.steps.length === 0) {
    errors.push(new ValidationError('steps', -1, 'pipeline', 'Pipeline must have at least one step'));
  }

  // Validate each step
  pipeline.steps.forEach((step, index) => {
    errors.push(...validateStep(step, index, schema));
  });

  return errors;
}

// Validate a single field for real-time feedback
export function validateSingleField(fieldName, value, fieldDef, stepIndex) {
  const errors = [];

  const errorMessage = validateField(fieldDef, value);
  if (errorMessage) {
    errors.push(new ValidationError(fieldName, stepIndex, fieldDef.field_type, errorMessage));
  }

  return errors;
}

// Display inline errors next to fields
export function displayFieldError(fieldName, stepIndex, error, container) {
  const errorId = `error-${stepIndex}-${fieldName}`;
  let errorEl = document.getElementById(errorId);

  if (errorEl) {
    errorEl.textContent = error.message;
    errorEl.style.display = 'block';
  } else {
    errorEl = document.createElement('span');
    errorEl.id = errorId;
    errorEl.className = 'field-error';
    errorEl.textContent = error.message;
    container.appendChild(errorEl);
  }
}

export function clearFieldError(fieldName, stepIndex, container) {
  const errorId = `error-${stepIndex}-${fieldName}`;
  const errorEl = document.getElementById(errorId);
  if (errorEl) {
    errorEl.style.display = 'none';
  }
}

// Display summary of all validation errors
export function displayValidationSummary(errors, container) {
  container.innerHTML = '';

  if (errors.length === 0) {
    container.style.display = 'none';
    return;
  }

  const summary = document.createElement('div');
  summary.className = 'validation-summary';

  const title = document.createElement('h3');
  title.textContent = `Please fix ${errors.length} error(s):`;
  summary.appendChild(title);

  const list = document.createElement('ul');
  errors.forEach(error => {
    const li = document.createElement('li');
    li.textContent = error.message;
    if (error.stepIndex >= 0) {
      li.textContent += ` (Step ${error.stepIndex + 1})`;
    }
    list.appendChild(li);
  });
  summary.appendChild(list);

  container.appendChild(summary);
  container.style.display = 'block';
}

export function highlightInvalidFields(errors) {
  // Clear all previous highlights
  document.querySelectorAll('.field-error').forEach(el => el.classList.remove('has-error'));

  // Add error class to invalid fields
  errors.forEach(error => {
    if (error.stepIndex >= 0) {
      const fieldId = `field-${error.stepIndex}-${error.fieldName}`;
      const fieldEl = document.getElementById(fieldId);
      if (fieldEl) {
        fieldEl.classList.add('field-error');
        fieldEl.classList.add('has-error');
      }
    }
  });
}

export function clearFieldHighlights() {
  document.querySelectorAll('.field-error').forEach(el => {
    el.classList.remove('field-error', 'has-error');
  });
}

const activeValidators = {};

export function registerValidator(fieldName, stepIndex, validatorFn) {

## Critical Validation Issue to Address

### Current Problem
The pipeline editor allows saving pipelines with completely empty fields, which is a critical data integrity issue. This happens because:

1. **Minimal Validation**: Only `validateMemorySteps()` exists, which only checks memory_agent checkboxes
2. **No Required Field Validation**: Text, textarea, select, and combobox fields can be empty
3. **No Empty Pipeline Check**: Pipelines with no steps can be created
4. **No Agent Type Validation**: Steps without an agent type are accepted
5. **No Client-Side Feedback**: Users only see errors via `alert()` after attempting save
6. **No Schema-Based Validation**: The schema has `required` flags but they're not enforced

### Impact
- Users can create invalid pipelines that will fail at runtime
- Data quality issues in the database
- Poor user experience (surprise failures)
- Difficult to debug pipeline execution errors
- Potential for system instability

### Solution in Refactoring

The refactoring addresses this by implementing:

1. **Schema-Based Field Validators**:
   - Each field type has its own validator function
   - Validators check `required` flag from schema
   - Validators handle edge cases (null, undefined, whitespace)

2. **Comprehensive Step Validation**:
   - Validates agent type is selected
   - Validates all required fields have values
   - Special validation for memory_agent
   - Returns specific, actionable error messages

3. **Pipeline-Level Validation**:
   - Ensures at least one step exists
   - Validates all steps in sequence
   - Collects and reports all errors

4. **Real-Time Validation**:
   - Validate fields on blur/input
   - Show inline error messages
   - Clear errors when fixed
   - Visual highlighting of invalid fields

5. **Improved UX**:
   - Validation summary at top of form
   - Step numbers in error messages
   - Inline errors next to fields
   - Clear, specific error messages
   - Save button disabled when invalid (optional)

### Validation Test Coverage

The refactoring includes comprehensive tests:

- **Unit Tests**: Each field validator tested individually
- **Integration Tests**: Validation flow through the editor
- **E2E Tests**: Full user journeys with validation
- **Edge Cases**: Null, undefined, whitespace, empty arrays
- **Error Scenarios**: Multiple errors, mixed valid/invalid

### Example Scenarios Fixed

**Before** (Current Behavior):
1. User creates pipeline
2. Adds step
3. Leaves agent type unselected
4. Leaves all fields empty
5. Clicks "Save pipeline"
6. Pipeline saves with empty values (BUG!)
7. Later fails during execution

**After** (Refactored Behavior):
1. User creates pipeline
2. Adds step
3. Selects agent type → fields render
4. Leaves fields empty → real-time validation shows errors
5. Clicks "Save pipeline" → validation summary shows all errors
6. User cannot save until all required fields are filled
7. Pipeline saves only when valid

### Backend Validation Note

While this refactoring focuses on client-side validation for better UX, backend validation should also be added in `mee6/web/routes/pipelines.py`:

1. Add validation in `create_pipeline()` and `update_pipeline()` endpoints
2. Check schema for required fields before saving
3. Return 400 Bad Request with validation errors if invalid
4. Use pydantic models with validation rules
5. Ensure data integrity at the database level

This provides defense in depth and prevents invalid data even if client-side validation is bypassed.
