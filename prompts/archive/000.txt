## Project overview

**Name:** mee6
**Description:** A personal AI assistant that automates life tasks using LLMs.
**Runtime:** Python 3.12, managed with `uv`, running in Docker.
**LLM backends:** Anthropic (Claude) and Ollama (external, already running on the user's
  home server вЂ” mee6 only needs to connect to it via URL, it does NOT run ollama itself).
**Agent framework:** agntrick + agntrick-whatsapp (published to PyPI by jeancsil).
**Interface:** A web UI where the user can configure and trigger scheduled tasks.

The first agent feature to scaffold is:
**school-monitor** вЂ” reads kids' school app, extracts calendar events, creates Google
Calendar entries, and sends WhatsApp notifications.

## Requirements per file

### pyproject.toml
- `[build-system]` with hatchling
- Python = ">=3.12"
- Dependencies:
  - `agntrick`
  - `agntrick-whatsapp`
  - `anthropic`
  - `ollama`                         # client only, no server
  - `fastapi>=0.115`
  - `uvicorn[standard]`
  - `jinja2`
  - `python-multipart`               # for form submissions in the web UI
  - `pydantic-settings`
  - `python-dotenv`
  - `google-auth`, `google-api-python-client`
  - `httpx`
  - `apscheduler>=4.0`
- Dev deps: `pytest`, `pytest-asyncio`, `ruff`, `mypy`

### .agntrick.yaml
- Provider: `anthropic`, model `claude-haiku-4-5-20251001`, temperature 0.2
- Add a comment showing how to switch to ollama by changing provider/base_url

### Dockerfile
- Multi-stage: builder (python:3.12-slim + uv) в†’ runtime (python:3.12-slim)
- Builder: `COPY pyproject.toml uv.lock ./` then `uv sync --frozen --no-dev`
- Runtime: copy venv + source, non-root user `mee6`
- Expose port `8080`
- Entrypoint: `uvicorn mee6.web.app:app --host 0.0.0.0 --port 8080`
  (the FastAPI app also starts the APScheduler on startup)

### docker-compose.yml
- Single service: `mee6`
- Builds from local Dockerfile
- `env_file: .env`
- Ports: `8080:8080`
- Volumes: `./data:/app/data` (for persistent scheduler state / SQLite)
- Restart: `unless-stopped`
- NO ollama service вЂ” ollama is external and accessed via `OLLAMA_BASE_URL` in config

### config.py
Settings via pydantic-settings (all from env):
- `ANTHROPIC_API_KEY: str`
- `OLLAMA_BASE_URL: str`              # e.g. http://192.168.1.10:11434
- `OLLAMA_DEFAULT_MODEL: str = "llama3"`
- `SCHOOL_APP_URL: str`
- `SCHOOL_APP_USERNAME: str`
- `SCHOOL_APP_PASSWORD: str`
- `NOTIFY_PHONE_NUMBER: str`          # E.164, e.g. +34612345678
- `GOOGLE_CALENDAR_ID: str`
- `GOOGLE_CREDENTIALS_FILE: str = "/app/data/credentials.json"`
- Plus whatever agntrick-whatsapp requires (discovered during inspection)

### web/app.py
- FastAPI app factory function `create_app() -> FastAPI`
- On startup: initialize APScheduler, load persisted triggers from SQLite, register jobs
- On shutdown: gracefully stop the scheduler
- Mount Jinja2 templates and static files
- Include routers from `web/routes/`

### web/routes/triggers.py
Endpoints for managing scheduled triggers:
- `GET /triggers` вЂ” list all configured triggers (render triggers.html)
- `POST /triggers` вЂ” create a new trigger (form: agent name, cron expression, enabled)
- `POST /triggers/{id}/toggle` вЂ” enable/disable a trigger
- `DELETE /triggers/{id}` вЂ” remove a trigger
- `POST /triggers/{id}/run-now` вЂ” manually fire a trigger immediately

### web/templates/
Keep the UI minimal but functional:
- `base.html`: navigation with links to Dashboard and Triggers; include a small status
  indicator showing how many jobs are active
- `dashboard.html`: table of recent task runs with agent name, timestamp, status
  (success/fail), and a short result summary
- `triggers.html`: table of all triggers with their cron schedule, enabled toggle,
  run-now button, and delete button; plus a simple form to add a new trigger
- Use plain HTML + minimal inline CSS (no frameworks, no build step needed)

### scheduler/engine.py
- Initialize `AsyncIOScheduler` from APScheduler
- Persist job store to SQLite at `/app/data/scheduler.db`
- Expose `add_trigger(agent_name, cron_expr)`, `remove_trigger(job_id)`,
  `toggle_trigger(job_id, enabled)`, `run_now(job_id)` functions
- On each trigger fire: look up the task function from `scheduler/registry.py` and run it

### scheduler/registry.py
- A dict mapping agent name strings to their task coroutine functions
- Example: `{"school-monitor": tasks.school_events.run_school_monitor}`
- New agents are registered here as they are added to the project

### agent.py (school_monitor)
- Use the EXACT decorator and base class API confirmed during inspection
- system_prompt: instructs the LLM to extract school events from raw HTML/text
  and return structured JSON (title, date, time, description, type)
- local_tools(): three StructuredTool stubs:
  1. `fetch_school_app` вЂ” fetches raw content from school app
  2. `create_calendar_event` вЂ” takes title, date, time, description
  3. `send_whatsapp_notification` вЂ” takes phone, message

### tasks/school_events.py
Async function `run_school_monitor() -> dict`:
1. Run the school_monitor agent
2. Parse JSON output into event list
3. For each event: create calendar entry + send WhatsApp notification
4. Return a result summary dict (used by dashboard to show last run status)
